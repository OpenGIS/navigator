# Task: Document First Sync Review

Perform a full semantic review of the Navigator project to verify that
documentation, tests, and screenshots are in sync with the Document First
process.

---

## What to check

### 1. Every guide doc has a corresponding test spec

For each file matching `docs/guide/*.md` (and `docs/guide/features/*.md`):
- There must be a corresponding `tests/e2e/*.spec.js` (or `tests/e2e/features/*.spec.js`)
- Developer docs in `docs/dev/` do not need test specs

### 2. Test describe blocks match doc headings

For each `tests/e2e/*.spec.js`:
- Open both the spec file and the corresponding guide doc
- Check that each `test.describe` name corresponds to a heading in the doc
- Report any describe blocks that have no matching heading (orphaned tests)
- Report any headings that describe runtime behaviour but have no describe block (untested behaviour)

Note: Heading names and describe names don't need to be identical — use judgement. A heading
"View persistence" and a describe block "useMap / View persistence" are clearly aligned.

### 3. Screenshot specs match doc image references

For each `tests/e2e/screenshots/*.spec.js`:
- Open the corresponding `docs/guide/*.md`
- Check that every `![...](../../assets/screenshots/docs/name/filename.png)` image reference
  in the doc is captured by a `page.screenshot({ path: ... })` call in the spec
- Report any doc images not generated by any spec (will be broken in docs)
- Report any spec screenshots not referenced in any doc (orphaned screenshots)

### 4. Screenshot files exist on disk

For each image referenced in any `docs/guide/*.md`:
- Verify the file exists at the stated path under `assets/screenshots/`
- Report any missing files (docs will show broken images)

You can also run `npm run check:sync` for the mechanical checks (file existence,
image paths). This task covers the semantic layer on top of that.

---

## How to run

Read this file, then:

1. Run `npm run check:sync` — fix any reported issues first
2. List all `docs/guide/*.md` files (and `docs/guide/features/*.md`)
3. For each guide doc, open the doc and its corresponding spec side by side
4. Work through the checklist above for each pair
5. Report a summary: ✅ aligned / ⚠️ gaps found, with specifics

---

## Expected output

A report like:

```
docs/guide/core.md  ↔  tests/e2e/core.spec.js
  ✅ useMap / Initialising the map  → describe block found
  ✅ useMap / View persistence      → describe block found
  ...

Screenshots
  ✅ assets/screenshots/docs/core/first-load.png  → exists, referenced in docs, generated by spec
  ✅ assets/screenshots/docs/core/panel.png        → exists, referenced in docs, generated by spec
  ✅ assets/screenshots/docs/core/mobile.png       → exists, referenced in docs, generated by spec
```
