# Extending Navigator

Navigator is designed to be extended. The `create()` / `mount()` API, event emitter, lifecycle callbacks, config-driven buttons, and plugin system give consumers multiple levels of customisation — from simple one-liners to full third-party plugins.

---

## The `create()` / `mount()` API

`Navigator.create()` returns a `NavigatorInstance` without mounting it. This gives you a window to configure the Vue app, register event listeners, or install plugins before the UI renders.

```js
import Navigator from '@ogis/navigator';
import '@ogis/navigator/navigator.css';

const nav = Navigator.create({
  id: 'my-map',
  locale: 'fr',
  mapOptions: { center: [-128.0094, 50.6539], zoom: 12 },
});

// Configure before mount
nav.on('map:ready', ({ map }) => {
  map.addSource('markers', { type: 'geojson', data: myGeoJSON });
});

nav.mount();
```

### NavigatorInstance

The object returned by `Navigator.create()`:

| Property / Method | Description |
|-------------------|-------------|
| `app` | The Vue 3 `App` instance (not yet mounted) |
| `id` | The instance identifier string |
| `mount()` | Mount the app to the DOM. Returns `this` for chaining. |
| `unmount()` | Unmount and clean up. Removes the event emitter. |
| `on(event, fn)` | Register an event listener |
| `once(event, fn)` | Register a one-time listener |
| `off(event, fn)` | Remove a listener (or all listeners for an event if `fn` is omitted) |
| `emit(event, ...args)` | Emit an event |

Because `app` is exposed before mount, you can call any [Vue app API](https://vuejs.org/api/application.html):

```js
const nav = Navigator.create({ id: 'my-map' });

nav.app.provide('analytics', myAnalyticsService);
nav.app.component('CustomWidget', MyWidget);
nav.app.use(MyVuePlugin);

nav.mount();
```

## Event Emitter

Every Navigator instance has a built-in event emitter. Use it to react to internal state changes without reaching into Vue internals.

### Built-in Events

| Event | Payload | Fired when |
|-------|---------|------------|
| `map:ready` | `{ map, instanceId }` | MapLibre map finishes loading |
| `view:change` | `{ center: { lat, lng }, zoom }` | Map view changes (throttled to 1 s) |
| `theme:change` | `'light'` \| `'dark'` | Resolved theme changes (toggle or system) |
| `panel:change` | `panelId` (string) | Active side-panel tab changes |

### Usage

```js
const nav = Navigator.create({ id: 'my-map' });

nav.on('map:ready', ({ map, instanceId }) => {
  console.log('Map loaded for', instanceId);
});

nav.on('view:change', ({ center, zoom }) => {
  console.log(`View: ${center.lat}, ${center.lng} @ zoom ${zoom}`);
});

nav.on('theme:change', (theme) => {
  document.querySelector('.my-widget').dataset.theme = theme;
});

nav.mount();
```

### Custom Events

You can also emit and listen for your own events:

```js
nav.on('my-plugin:update', (data) => { /* ... */ });
nav.emit('my-plugin:update', { count: 42 });
```

## Lifecycle Callbacks

For simple use cases, lifecycle callbacks in the config are a convenient alternative to the event emitter. Each callback maps directly to a built-in event.

```js
Navigator.create({
  id: 'my-map',
  onMapReady: ({ map, instanceId }) => {
    map.addLayer({ id: 'custom', type: 'circle', source: 'points' });
  },
  onViewChange: ({ center, zoom }) => {
    updateUrlParams(center, zoom);
  },
  onThemeChange: (theme) => {
    applyThemeToExternalWidgets(theme);
  },
  onPanelChange: (panelId) => {
    analytics.track('panel_view', { panel: panelId });
  },
}).mount();
```

| Callback | Event | Payload |
|----------|-------|---------|
| `onMapReady` | `map:ready` | `{ map, instanceId }` |
| `onViewChange` | `view:change` | `{ center: { lat, lng }, zoom }` |
| `onThemeChange` | `theme:change` | `'light'` \| `'dark'` |
| `onPanelChange` | `panel:change` | `panelId` (string) |

Callbacks are registered as event listeners during `create()`, so they can be removed later with `nav.off()` if needed.

## Config-Driven Buttons and Panels

Add custom buttons to the top navigation bar, custom tabs to the side panel, or both — entirely through the config.

### Custom Buttons

Buttons appear in the top navigation bar. They can trigger actions via `onClick` and optionally open a panel tab.

```js
Navigator.create({
  id: 'my-map',
  buttons: [
    {
      id: 'measure',
      icon: 'ruler',
      label: 'Measure',
      position: 'end',       // 'start' | 'middle' | 'end' (default: 'end')
      onClick: ({ map, instanceId }) => {
        console.log('Measure clicked');
      },
    },
  ],
}).mount();
```

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `id` | `string` | ✓ | Unique button identifier |
| `icon` | `string` | * | Icon name from the SVG sprite |
| `label` | `string` | * | Button label text |
| `position` | `string` | | Toolbar slot: `'start'`, `'middle'`, or `'end'` (default) |
| `component` | `Component` | | Vue component to render instead of the default icon button |
| `props` | `object` | | Props to pass to the Vue component |
| `onClick` | `function` | | Click handler, receives `{ map, instanceId }` |
| `panel` | `object` | | Panel config — adds a side-panel tab (see below) |

\* Required unless `component` is provided.

#### Vue component button

For full control over a button's appearance and behaviour, provide a Vue component instead of `icon` / `label`. The component is rendered directly in the navbar, inside Navigator's Vue app tree — giving it full access to dependency injection and reactivity.

```js
import HomeButton from './components/HomeButton.vue';

Navigator.create({
  id: 'my-map',
  buttons: [
    {
      id: 'home',
      position: 'middle',
      component: HomeButton,
    },
  ],
}).mount();
```

Inside your button component, use `inject('navigatorId')` and `getMapInstance()` to access the Navigator context:

```vue
<!-- HomeButton.vue — a centered navbar button that flies to a home location -->
<script setup>
import { inject } from 'vue';
import { getMapInstance } from '@ogis/navigator';

const instanceId = inject('navigatorId', 'navigator');

const goHome = () => {
  const map = getMapInstance(instanceId);
  if (map) map.flyTo({ center: [-128.0094, 50.6539], zoom: 12 });
};
</script>

<template>
  <button
    type="button"
    class="border-0 bg-transparent text-white d-flex flex-column align-items-center"
    @click="goHome"
  >
    <svg width="40" height="40" fill="currentColor">
      <use href="#house-fill" />
    </svg>
    <small>Home</small>
  </button>
</template>
```

When `component` is provided, `icon`, `label`, and `onClick` are ignored — the component manages its own rendering and event handling.

> **Tip — Replacing the entire navbar:** For maximum flexibility, use `nav.app.component()` between `create()` and `mount()` to override any internal Navigator component. This lets you supply a completely custom navbar if needed.

A button with a `panel` property acts as both a toolbar button and a panel tab. Clicking it opens the side panel with custom content.

#### Vue component panel (preferred)

The recommended way to provide panel content is with a Vue component. The component is rendered inside Navigator's Vue app tree, giving it full access to Vue's dependency injection and reactivity.

```js
import MeasurePanel from './components/MeasurePanel.vue';

Navigator.create({
  id: 'my-map',
  buttons: [
    {
      id: 'measure',
      icon: 'ruler',
      label: 'Measure',
      panel: {
        title: 'Measurement Tool',
        component: MeasurePanel,
        props: { units: 'metric' },
      },
    },
  ],
}).mount();
```

Inside the component, use `inject('navigatorId')` and `getMapInstance()` to access the Navigator context:

```vue
<!-- MeasurePanel.vue -->
<script setup>
import { inject } from 'vue';
import { getMapInstance } from '@ogis/navigator';

const props = defineProps({ units: String });

const instanceId = inject('navigatorId', 'navigator');
const map = getMapInstance(instanceId);
</script>

<template>
  <div class="p-3">
    <h5>Measurement Tool</h5>
    <p>Units: {{ units }}</p>
    <p>Draw on the map to measure distances.</p>
  </div>
</template>
```

#### DOM render function

For simple content or non-Vue integrations, use a `render` function that receives a DOM container:

```js
buttons: [
  {
    id: 'info',
    icon: 'info-circle',
    label: 'Info',
    panel: {
      title: 'Location Info',
      render: (container, { map, instanceId }) => {
        container.innerHTML = '<h5>Details</h5><p>Custom content here.</p>';
      },
    },
  },
]
```

### Custom Panels

Use the `panels` config when you want a side-panel tab without a toolbar button.

#### Vue component (preferred)

```js
import InfoPanel from './components/InfoPanel.vue';

Navigator.create({
  id: 'my-map',
  panels: [
    {
      id: 'info',
      icon: 'info-circle',
      title: 'Info',
      component: InfoPanel,
    },
  ],
}).mount();
```

#### DOM render function

```js
Navigator.create({
  id: 'my-map',
  panels: [
    {
      id: 'info',
      icon: 'info-circle',
      title: 'Info',
      render: (container, { map, instanceId }) => {
        container.innerHTML = '<h5>Details</h5><p>Custom content here.</p>';
      },
    },
  ],
}).mount();
```

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `id` | `string` | ✓ | Unique panel identifier |
| `icon` | `string` | ✓ | Icon name from the SVG sprite |
| `title` | `string` | ✓ | Tab label |
| `component` | `Component` | | Vue component to render (preferred) |
| `props` | `object` | | Props to pass to the Vue component |
| `render` | `function` | | Called with `(container, { map, instanceId })` to populate content |

> Provide either `component` or `render`. If both are present, `component` takes precedence.

The `render` function receives a DOM element (`container`) and a context object. Navigator clears the container and calls `render` each time the tab is activated.

> **Buttons vs Panels:** Use `buttons` when you need a toolbar button (with an optional panel). Use `panels` when you only need a panel tab. Both can be combined.

### Theming

Custom panel content automatically inherits the current theme via Bootstrap CSS variables. Use `var(--bs-body-bg)`, `var(--bs-body-color)`, etc. for consistent light/dark support.

## Plugin System

Plugins are reusable extensions that receive the Navigator context and can register event listeners, provide data to the Vue tree, or add map layers.

### Writing a Plugin

A plugin is any object with an `install` method:

```js
const MarkerPlugin = {
  install({ app, instanceId, on, off, emit }, options) {
    // Listen for map ready
    on('map:ready', ({ map }) => {
      map.addSource('markers', {
        type: 'geojson',
        data: options.data,
      });
      map.addLayer({
        id: 'markers-layer',
        type: 'circle',
        source: 'markers',
        paint: { 'circle-radius': 6, 'circle-color': '#e74c3c' },
      });
      emit('markers:loaded', { count: options.data.features.length });
    });
  },
};
```

### Plugin Context

The `install` function receives a context object and optional options:

| Property | Description |
|----------|-------------|
| `app` | The Vue 3 `App` instance |
| `instanceId` | The Navigator instance ID |
| `on(event, fn)` | Register an event listener |
| `off(event, fn)` | Remove an event listener |
| `emit(event, ...args)` | Emit an event |

### Global Plugins

Register a plugin globally so it is installed on every instance:

```js
Navigator.use(MarkerPlugin, { data: myGeoJSON });

// Both instances receive the plugin
Navigator.create({ id: 'map-a' }).mount();
Navigator.create({ id: 'map-b' }).mount();
```

`Navigator.use()` returns `Navigator` for chaining:

```js
Navigator
  .use(MarkerPlugin, { data: pointsGeoJSON })
  .use(AnalyticsPlugin, { trackingId: 'UA-XXX' });
```

### Per-Instance Plugins

Pass plugins in the config to install only on a specific instance:

```js
Navigator.create({
  id: 'map-a',
  plugins: [
    { install: ({ on }) => on('map:ready', () => console.log('map-a ready')) },
  ],
}).mount();
```

## Accessing the Map

The MapLibre map instance is available via `getMapInstance`:

```js
import { getMapInstance } from '@ogis/navigator';

const map = getMapInstance('my-map');
if (map) {
  map.flyTo({ center: [-128.0094, 50.6539], zoom: 14 });
}
```

The map is `null` until MapLibre finishes loading. Use the `map:ready` event or `onMapReady` callback instead of polling:

```js
nav.on('map:ready', ({ map }) => {
  // map is guaranteed to be loaded here
  map.addSource('route', { type: 'geojson', data: routeGeoJSON });
});
```

## Controlling the Panel

Components rendered inside the Navigator tree (via the `component` config in buttons or panels) can programmatically control the side panel using `useUI`:

```js
import { useUI } from '@ogis/navigator';

const { setActivePanel, openPanel, closePanel } = useUI();

// Switch to a panel tab and open the panel
setActivePanel('my-panel-id');
openPanel();
```

`useUI()` relies on `inject('navigatorId')` internally, so it only works inside a Vue component that is part of Navigator's tree — such as a custom button or panel component passed through the config.

## Persisting Custom State

Use `useStorage` from within the source tree to persist state to `localStorage`, automatically scoped to the Navigator instance:

```js
import { useStorage } from '@/composables/useStorage';

// Stored as "navigator_my-feature_{instanceId}"
const state = useStorage('my-feature', {
  enabled: false,
  lastValue: null,
});
```

See [Instances](./2.instances.md) for the full storage key convention.

## Summary

| Goal | API |
|------|-----|
| React to map load | `onMapReady` callback or `map:ready` event |
| Track view changes | `onViewChange` callback or `view:change` event |
| Respond to theme toggle | `onThemeChange` callback or `theme:change` event |
| Add a toolbar button | `buttons` array in config |
| Add a Vue component button | `buttons` with `component` property |
| Add a toolbar button with panel | `buttons` with `panel` property |
| Add a panel tab (no button) | `panels` array in config |
| Render a Vue component in a panel | `component` property in panel config |
| Write a reusable extension | Plugin with `install()` method |
| Configure Vue before mount | Access `nav.app` between `create()` and `mount()` |
| Replace the entire navbar | `nav.app.component()` before `mount()` |
| Access the map externally | `getMapInstance(id)` or `map:ready` event |
| Control the panel from a component | `useUI()` export |
