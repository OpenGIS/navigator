# Extending Navigator

Navigator is designed to be extended. The `create()` / `mount()` API, event emitter, lifecycle callbacks, config-driven buttons, and plugin system give consumers multiple levels of customisation ŌĆö from simple one-liners to full third-party plugins.

---

## The `create()` / `mount()` API

`Navigator.create()` returns a `NavigatorInstance` without mounting it. This gives you a window to configure the Vue app, register event listeners, or install plugins before the UI renders.

```js
import Navigator from '@ogis/navigator';
import '@ogis/navigator/navigator.css';

const nav = Navigator.create({
  id: 'my-map',
  locale: 'fr',
  mapOptions: { center: [-128.0094, 50.6539], zoom: 12 },
});

// Configure before mount
nav.on('map:ready', ({ map }) => {
  map.addSource('markers', { type: 'geojson', data: myGeoJSON });
});

nav.mount();
```

### NavigatorInstance

The object returned by `Navigator.create()`:

| Property / Method | Description | Since |
|-------------------|-------------|-------|
| `app` | The Vue 3 `App` instance (not yet mounted) | 1.0.0 |
| `id` | The instance identifier string | 1.0.0 |
| `mount()` | Mount the app to the DOM. Returns `this` for chaining. | 1.0.0 |
| `unmount()` | Unmount and clean up. Emits `destroy`, runs plugin cleanups, then unmounts. | 1.0.0 |
| `on(event, fn)` | Register an event listener | 1.0.0 |
| `once(event, fn)` | Register a one-time listener | 1.0.0 |
| `off(event, fn)` | Remove a listener (or all listeners for an event if `fn` is omitted) | 1.0.0 |
| `emit(event, ...args)` | Emit an event | 1.0.0 |

Because `app` is exposed before mount, you can call any [Vue app API](https://vuejs.org/api/application.html):

```js
const nav = Navigator.create({ id: 'my-map' });

nav.app.provide('analytics', myAnalyticsService);
nav.app.component('CustomWidget', MyWidget);
nav.app.use(MyVuePlugin);

nav.mount();
```

## Event Emitter

Every Navigator instance has a built-in event emitter. Use it to react to internal state changes without reaching into Vue internals.

### Built-in Events

| Event | Payload | Fired when | Since |
|-------|---------|------------|-------|
| `map:ready` | `{ map, instanceId }` | MapLibre map finishes loading | 1.0.0 |
| `view:change` | `{ center: { lat, lng }, zoom }` | Map view changes (throttled to 1 s) | 1.0.0 |
| `theme:change` | `'light'` \| `'dark'` | Resolved theme changes (toggle or system) | 1.0.13 |
| `panel:change` | `panelId` (string) | Active side-panel tab changes | 1.0.16 |
| `destroy` | *(none)* | Navigator instance is about to unmount | 1.0.21 |

### Usage

```js
const nav = Navigator.create({ id: 'my-map' });

nav.on('map:ready', ({ map, instanceId }) => {
  console.log('Map loaded for', instanceId);
});

nav.on('view:change', ({ center, zoom }) => {
  console.log(`View: ${center.lat}, ${center.lng} @ zoom ${zoom}`);
});

nav.on('theme:change', (theme) => {
  document.querySelector('.my-widget').dataset.theme = theme;
});

nav.mount();
```

### Custom Events

You can also emit and listen for your own events:

```js
nav.on('my-plugin:update', (data) => { /* ... */ });
nav.emit('my-plugin:update', { count: 42 });
```

## Lifecycle Callbacks

For simple use cases, lifecycle callbacks in the config are a convenient alternative to the event emitter. Each callback maps directly to a built-in event.

```js
Navigator.create({
  id: 'my-map',
  onMapReady: ({ map, instanceId }) => {
    map.addLayer({ id: 'custom', type: 'circle', source: 'points' });
  },
  onViewChange: ({ center, zoom }) => {
    updateUrlParams(center, zoom);
  },
  onThemeChange: (theme) => {
    applyThemeToExternalWidgets(theme);
  },
  onPanelChange: (panelId) => {
    analytics.track('panel_view', { panel: panelId });
  },
}).mount();
```

| Callback | Event | Payload | Since |
|----------|-------|---------|-------|
| `onMapReady` | `map:ready` | `{ map, instanceId }` | 1.0.0 |
| `onViewChange` | `view:change` | `{ center: { lat, lng }, zoom }` | 1.0.0 |
| `onThemeChange` | `theme:change` | `'light'` \| `'dark'` | 1.0.13 |
| `onPanelChange` | `panel:change` | `panelId` (string) | 1.0.16 |

Callbacks are registered as event listeners during `create()`, so they can be removed later with `nav.off()` if needed.

## Config-Driven Buttons and Panels

Add custom buttons to the top navigation bar, custom tabs to the side panel, or both ŌĆö entirely through the config.

### Custom Buttons

Buttons appear in the top navigation bar. They can trigger actions via `onClick` and optionally open a panel tab.

```js
Navigator.create({
  id: 'my-map',
  buttons: [
    {
      id: 'measure',
      icon: 'ruler',
      label: 'Measure',
      position: 'end',       // 'start' | 'middle' | 'end' (default: 'end')
      onClick: ({ map, instanceId }) => {
        console.log('Measure clicked');
      },
    },
  ],
}).mount();
```

| Property | Type | Required | Description | Since |
|----------|------|----------|-------------|-------|
| `id` | `string` | Ō£ō | Unique button identifier | 1.0.0 |
| `icon` | `string` | * | Icon name from the SVG sprite | 1.0.0 |
| `label` | `string` | * | Button label text | 1.0.0 |
| `position` | `string` | | Toolbar slot: `'start'`, `'middle'`, or `'end'` (default) | 1.0.0 |
| `component` | `Component` | | Vue component to render instead of the default icon button | 1.0.16 |
| `props` | `object` | | Props to pass to the Vue component | 1.0.16 |
| `onClick` | `function` | | Click handler, receives `{ map, instanceId }` | 1.0.0 |
| `panel` | `object` | | Panel config ŌĆö adds a side-panel tab (see below) | 1.0.0 |

\* Required unless `component` is provided.

#### Vue component button

For full control over a button's appearance and behaviour, provide a Vue component instead of `icon` / `label`. The component is rendered directly in the navbar, inside Navigator's Vue app tree ŌĆö giving it full access to dependency injection and reactivity.

```js
import HomeButton from './components/HomeButton.vue';

Navigator.create({
  id: 'my-map',
  buttons: [
    {
      id: 'home',
      position: 'middle',
      component: HomeButton,
    },
  ],
}).mount();
```

Inside your button component, use `useMap()` to access the MapLibre instance:

```vue
<!-- HomeButton.vue ŌĆö a centered navbar button that flies to a home location -->
<script setup>
import { useMap } from '@ogis/navigator';

const { map } = useMap();

const goHome = () => {
  if (map) map.flyTo({ center: [-128.0094, 50.6539], zoom: 12 });
};
</script>

<template>
  <button
    type="button"
    class="border-0 bg-transparent text-white d-flex flex-column align-items-center"
    @click="goHome"
  >
    <svg width="40" height="40" fill="currentColor">
      <use href="#house-fill" />
    </svg>
    <small>Home</small>
  </button>
</template>
```

When `component` is provided, `icon`, `label`, and `onClick` are ignored ŌĆö the component manages its own rendering and event handling.

> **Tip ŌĆö Replacing the entire navbar:** For maximum flexibility, use `nav.app.component()` between `create()` and `mount()` to override any internal Navigator component. This lets you supply a completely custom navbar if needed.

A button with a `panel` property acts as both a toolbar button and a panel tab. Clicking it opens the side panel with custom content.

#### Vue component panel (preferred)

The recommended way to provide panel content is with a Vue component. The component is rendered inside Navigator's Vue app tree, giving it full access to Vue's dependency injection and reactivity.

```js
import MeasurePanel from './components/MeasurePanel.vue';

Navigator.create({
  id: 'my-map',
  buttons: [
    {
      id: 'measure',
      icon: 'ruler',
      label: 'Measure',
      panel: {
        title: 'Measurement Tool',
        component: MeasurePanel,
        props: { units: 'metric' },
      },
    },
  ],
}).mount();
```

Inside the component, use `useMap()` to access the MapLibre instance:

```vue
<!-- MeasurePanel.vue -->
<script setup>
import { useMap } from '@ogis/navigator';

const props = defineProps({ units: String });

const { map } = useMap();
</script>

<template>
  <div class="p-3">
    <h5>Measurement Tool</h5>
    <p>Units: {{ units }}</p>
    <p>Draw on the map to measure distances.</p>
  </div>
</template>
```

#### DOM render function

For simple content or non-Vue integrations, use a `render` function that receives a DOM container:

```js
buttons: [
  {
    id: 'info',
    icon: 'info-circle',
    label: 'Info',
    panel: {
      title: 'Location Info',
      render: (container, { map, instanceId }) => {
        container.innerHTML = '<h5>Details</h5><p>Custom content here.</p>';
      },
    },
  },
]
```

### Custom Panels

Use the `panels` config when you want a side-panel tab without a toolbar button.

#### Vue component (preferred)

```js
import InfoPanel from './components/InfoPanel.vue';

Navigator.create({
  id: 'my-map',
  panels: [
    {
      id: 'info',
      icon: 'info-circle',
      title: 'Info',
      component: InfoPanel,
    },
  ],
}).mount();
```

#### DOM render function

```js
Navigator.create({
  id: 'my-map',
  panels: [
    {
      id: 'info',
      icon: 'info-circle',
      title: 'Info',
      render: (container, { map, instanceId }) => {
        container.innerHTML = '<h5>Details</h5><p>Custom content here.</p>';
      },
    },
  ],
}).mount();
```

| Property | Type | Required | Description | Since |
|----------|------|----------|-------------|-------|
| `id` | `string` | Ō£ō | Unique panel identifier | 1.0.0 |
| `icon` | `string` | Ō£ō | Icon name from the SVG sprite | 1.0.0 |
| `title` | `string` | Ō£ō | Tab label | 1.0.0 |
| `component` | `Component` | | Vue component to render (preferred) | 1.0.16 |
| `props` | `object` | | Props to pass to the Vue component | 1.0.16 |
| `render` | `function` | | Called with `(container, { map, instanceId })` to populate content | 1.0.0 |

> Provide either `component` or `render`. If both are present, `component` takes precedence.

The `render` function receives a DOM element (`container`) and a context object. Navigator clears the container and calls `render` each time the tab is activated.

> **Buttons vs Panels:** Use `buttons` when you need a toolbar button (with an optional panel). Use `panels` when you only need a panel tab. Both can be combined.

### Theming

Custom panel content automatically inherits the current theme via Bootstrap CSS variables. Use `var(--bs-body-bg)`, `var(--bs-body-color)`, etc. for consistent light/dark support.

## Plugin System

Plugins are reusable extensions that receive the Navigator context and can register event listeners, provide data to the Vue tree, or add map layers.

### Writing a Plugin

A plugin is any object with an `install` method. The `install` method may optionally return a **cleanup function** that will be called when the Navigator instance is unmounted:

```js
const MarkerPlugin = {
  install({ app, instanceId, on, once, off, emit }, options) {
    // Listen for map ready (fires once, so use once())
    once('map:ready', ({ map }) => {
      map.addSource('markers', {
        type: 'geojson',
        data: options.data,
      });
      map.addLayer({
        id: 'markers-layer',
        type: 'circle',
        source: 'markers',
        paint: { 'circle-radius': 6, 'circle-color': '#e74c3c' },
      });
      emit('markers:loaded', { count: options.data.features.length });
    });

    // Return a cleanup function (optional)
    return () => {
      // Called when the Navigator instance is unmounted
    };
  },
};
```

### Plugin Context

The `install` function receives a context object and optional options:

| Property | Description | Since |
|----------|-------------|-------|
| `app` | The Vue 3 `App` instance | 1.0.0 |
| `instanceId` | The Navigator instance ID | 1.0.0 |
| `on(event, fn)` | Register an event listener | 1.0.0 |
| `once(event, fn)` | Register a one-time event listener | 1.0.21 |
| `off(event, fn)` | Remove an event listener | 1.0.0 |
| `emit(event, ...args)` | Emit an event | 1.0.0 |

If `install` returns a function, it will be called when the Navigator instance is unmounted (see [Plugin Cleanup](#plugin-cleanup) below).

### Global Plugins

Register a plugin globally so it is installed on every instance:

```js
Navigator.use(MarkerPlugin, { data: myGeoJSON });

// Both instances receive the plugin
Navigator.create({ id: 'map-a' }).mount();
Navigator.create({ id: 'map-b' }).mount();
```

`Navigator.use()` returns `Navigator` for chaining:

```js
Navigator
  .use(MarkerPlugin, { data: pointsGeoJSON })
  .use(AnalyticsPlugin, { trackingId: 'UA-XXX' });
```

### Per-Instance Plugins

Pass plugins in the config to install only on a specific instance:

```js
Navigator.create({
  id: 'map-a',
  plugins: [
    { install: ({ on }) => on('map:ready', () => console.log('map-a ready')) },
  ],
}).mount();
```

### Plugin Cleanup

Plugins can clean up resources (timers, geolocation watches, map layers) in two ways:

**1. Return a cleanup function from `install()`**

The returned function is called automatically when the Navigator instance is unmounted:

```js
const PollingPlugin = {
  install({ instanceId }) {
    const timerId = setInterval(() => { /* poll */ }, 5000);

    // Cleanup: called on unmount
    return () => clearInterval(timerId);
  },
};
```

**2. Listen for the `destroy` event**

The `destroy` event fires immediately before the instance unmounts. This is useful when the cleanup logic is set up after install (e.g. inside a `map:ready` handler):

```js
const LayerPlugin = {
  install({ on, once }) {
    let map = null;

    once('map:ready', ({ map: m }) => {
      map = m;
      map.addSource('my-source', { type: 'geojson', data: emptyGeoJSON });
      map.addLayer({ id: 'my-layer', type: 'circle', source: 'my-source' });
    });

    on('destroy', () => {
      if (map?.getLayer('my-layer')) map.removeLayer('my-layer');
      if (map?.getSource('my-source')) map.removeSource('my-source');
    });
  },
};
```

Both mechanisms can be combined. The `destroy` event fires first, then return-value cleanup functions run, then the Vue app unmounts.

## Accessing the Map

### From outside Vue ŌĆö `getMapInstance()`

The MapLibre map instance is available via `getMapInstance`:

```js
import { getMapInstance } from '@ogis/navigator';

const map = getMapInstance('my-map');
if (map) {
  map.flyTo({ center: [-128.0094, 50.6539], zoom: 14 });
}
```

### From inside Vue ŌĆö `useMap()`

Components rendered inside Navigator's Vue tree (via the `component` config) can use the `useMap()` composable, which resolves the instance ID automatically:

```js
import { useMap } from '@ogis/navigator';

const { map } = useMap();
if (map) {
  map.flyTo({ center: [-128.0094, 50.6539], zoom: 14 });
}
```

`useMap()` calls `inject('navigatorId')` internally, so it only works inside a Vue component's `<script setup>` that is part of Navigator's tree.

### Waiting for the map

The map is `null` until MapLibre finishes loading. Use the `map:ready` event or `onMapReady` callback instead of polling:

```js
nav.on('map:ready', ({ map }) => {
  // map is guaranteed to be loaded here
  map.addSource('route', { type: 'geojson', data: routeGeoJSON });
});
```

## Controlling the Panel

Components rendered inside the Navigator tree (via the `component` config in buttons or panels) can programmatically control the side panel using `useUI`:

```js
import { useUI } from '@ogis/navigator';

const { setActivePanel, openPanel, closePanel } = useUI();

// Switch to a panel tab and open the panel
setActivePanel('my-panel-id');
openPanel();
```

`useUI()` relies on `inject('navigatorId')` internally, so it only works inside a Vue component that is part of Navigator's tree ŌĆö such as a custom button or panel component passed through the config.

## Persisting Custom State

Use `useStorage` to persist state to `localStorage`, automatically scoped to the Navigator instance:

```js
import { useStorage } from '@ogis/navigator';

// Stored as "navigator_my-feature_{instanceId}"
const state = useStorage('my-feature', {
  enabled: false,
  lastValue: null,
});
```

Inside a Vue component's `<script setup>`, the instance ID is resolved automatically via `inject('navigatorId')`. In a plugin `install()` function (which runs outside Vue's setup context), pass the `instanceId` explicitly as the third argument:

```js
const MyPlugin = {
  install({ instanceId }) {
    const state = useStorage('my-feature', { count: 0 }, instanceId);
    // state is reactive and auto-persisted
  },
};
```

See [Instances](./2.instances.md) for the full storage key convention.

## Build Prerequisites

When using Vue Single File Components (`.vue` files) in your plugins, buttons, or panels, your Vite config must include `@vitejs/plugin-vue`:

```bash
npm install --save-dev @vitejs/plugin-vue
```

```js
// vite.config.js
import vue from '@vitejs/plugin-vue';

export default {
  plugins: [vue()],
};
```

Without this plugin, Vite cannot compile `.vue` files and the build will fail with an opaque transform error. If you only use plain `.js` plugins (no SFCs), this is not needed.
