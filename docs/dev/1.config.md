# Configuration

Complete reference for all `Navigator.create()` options.

---

## Navigator.create()

```js
import Navigator from '@ogis/navigator';
import '@ogis/navigator/navigator.css';

const nav = Navigator.create({
  id:         'my-map',   // string   — default: 'navigator'
  debug:      false,      // boolean  — default: false
  locale:     'fr',       // string   — default: null (browser language)
  messages:   { … },     // object   — default: {}
  mapOptions: { … },     // object   — default: {}
  buttons:    [ … ],     // array    — default: []
  panels:     [ … ],     // array    — default: []
  plugins:    [ … ],     // array    — default: []
  onMapReady:    fn,      // function — default: undefined
  onViewChange:  fn,      // function — default: undefined
  onThemeChange: fn,      // function — default: undefined
  onPanelChange: fn,      // function — default: undefined
});

nav.mount();
```

All options are optional. Calling `Navigator.create()` with no arguments is valid.

`Navigator.create()` returns a `NavigatorInstance` — a wrapper around the Vue application that exposes the event emitter and mount/unmount lifecycle:

```js
const nav = Navigator.create({ id: "my-map" });

// Configure before mounting
nav.on("map:ready", ({ map }) => console.log("loaded"));
nav.app.provide("myService", myService);

nav.mount();

// Later...
nav.unmount();
```

See [Extending](./8.extending.md) for full details on the `NavigatorInstance` API.

| Option                                  | Type       | Default       | Purpose                                       |
| --------------------------------------- | ---------- | ------------- | --------------------------------------------- |
| [`id`](#id)                             | `string`   | `'navigator'` | DOM mount target and storage namespace        |
| [`debug`](#debug)                       | `boolean`  | `false`       | Reserved for future debug tooling             |
| [`locale`](#locale)                     | `string`   | `null`        | Default UI language                           |
| [`messages`](#messages)                 | `object`   | `{}`          | Per-language label overrides                  |
| [`mapOptions`](#mapoptions)             | `object`   | `{}`          | MapLibre `Map` constructor options            |
| [`buttons`](#buttons)                   | `array`    | `[]`          | Custom toolbar buttons (with optional panels) |
| [`panels`](#panels)                     | `array`    | `[]`          | Custom panel tabs (no toolbar button)         |
| [`plugins`](#plugins)                   | `array`    | `[]`          | Per-instance plugins                          |
| [`onMapReady`](#lifecycle-callbacks)    | `function` | —             | Called when the map finishes loading          |
| [`onViewChange`](#lifecycle-callbacks)  | `function` | —             | Called on map moveend                         |
| [`onThemeChange`](#lifecycle-callbacks) | `function` | —             | Called when the theme changes                 |
| [`onPanelChange`](#lifecycle-callbacks) | `function` | —             | Called when the active panel changes          |

## `id`

**Type:** `string` **Default:** `'navigator'`

The `id` option serves two purposes:

1. **Mount target.** Navigator looks for a `<div id="{id}">` in the DOM. If one is not found, it creates one and appends it to `<body>`.
2. **Storage namespace.** All `localStorage` keys for this instance are suffixed with the `id`, so multiple instances on the same page do not collide.

```js
Navigator.create({ id: "my-map" }).mount();
```

| Instance `id`         | localStorage key for map view |
| --------------------- | ----------------------------- |
| `navigator` (default) | `navigator_view_navigator`    |
| `my-map`              | `navigator_view_my-map`       |
| `secondary`           | `navigator_view_secondary`    |

See [Instances](./2.instances.md) for full details on instance isolation and the storage key format.

## `debug`

> [!CAUTION]
> Not yet implemented!

**Type:** `boolean` **Default:** `false`

Enables debug mode. This option is reserved for future tooling (verbose logging, dev overlays, etc.) and currently has no visible effect.

```js
Navigator.create({ id: "my-map", debug: true }).mount();
```

## `locale`

**Type:** `string` **Default:** `null`

Sets the default UI language when the user has no stored preference. Accepts any [BCP 47](https://www.ietf.org/rfc/bcp/bcp47.txt) language subtag supported by Navigator.

```js
Navigator.create({ id: "my-map", locale: "fr" }).mount();
```

Language resolution order:

1. **Stored user preference** — explicit language saved by the user in Settings
2. **`locale` option** — the value passed here
3. **Browser language** — `navigator.language`, exact then base code
4. **English fallback** — `'en'`

| Code | Language |
| ---- | -------- |
| `en` | English  |
| `fr` | Français |

See [Locale](./5.locale.md) for the full locale API, available languages, and how to contribute a translation.

## `messages`

**Type:** `object` **Default:** `{}`

Overrides individual UI labels for any language without modifying the built-in translations. Keys are BCP 47 language codes; values are objects mapping translation keys to replacement strings.

```js
Navigator.create({
  id: "my-map",
  messages: {
    en: {
      "modal.welcome.title": "My Map",
      "modal.welcome.getStarted": "Explore",
    },
    fr: {
      "modal.welcome.title": "Ma carte",
      "modal.welcome.getStarted": "Explorer",
    },
  },
}).mount();
```

Custom messages are merged on top of the built-in translations — only the keys you supply are overridden. Any unspecified keys continue to use the built-in values.

See [Locale](./5.locale.md) for the full list of translation keys.

## `mapOptions`

**Type:** `object` **Default:** `{}`

Any [MapLibre `MapOptions`](https://maplibre.org/maplibre-gl-js/docs/API/type-aliases/MapOptions/) can be passed here. They are merged with Navigator's defaults:

| Default              | Value                                                              |
| -------------------- | ------------------------------------------------------------------ |
| `style`              | OpenFreeMap bright (`https://tiles.openfreemap.org/styles/bright`) |
| `attributionControl` | `true`                                                             |

User-supplied keys override the defaults. For example, to supply a custom style and initial view:

```js
Navigator.create({
  id: "my-map",
  mapOptions: {
    center: [-128.0094, 50.6539],
    zoom: 12,
    bearing: 45,
    style: "https://example.com/style.json",
  },
}).mount();
```

> **View persistence:** if the user has previously visited, their last-known center and zoom are restored from `localStorage` and take precedence over `center`/`zoom` in `mapOptions`. The stored view is cleared when the user navigates to a new location using the URL hash.

See [Map](./3.map.md) for the full map API including view persistence and programmatic control.

## `buttons`

**Type:** `array` **Default:** `[]`

Adds custom buttons to the top navigation bar. Each button can optionally open a panel tab with custom content.

```js
Navigator.create({
  id: "my-map",
  buttons: [
    {
      id: "measure",
      icon: "ruler",
      label: "Measure",
      position: "end",
      onClick: ({ map, instanceId }) => {
        /* ... */
      },
      panel: {
        title: "Measurement Tool",
        render: (container, { map, instanceId }) => {
          container.innerHTML = "<p>Draw on the map to measure.</p>";
        },
      },
    },
  ],
}).mount();
```

See [Extending — Config-Driven Buttons and Panels](./8.extending.md#config-driven-buttons-and-panels) for the full button and panel config reference.

## `panels`

**Type:** `array` **Default:** `[]`

Adds custom tabs to the side panel without requiring a toolbar button. Use this when you want panel content accessible from the tab bar but don't need a top-bar button.

```js
Navigator.create({
  id: 'my-map',
  panels: [
    {
      id: 'info',
      icon: 'info-circle',
      title: 'Info',
      render: (container, { map, instanceId }) => {
        container.innerHTML = '<h5>Details</h5><p>Custom content here.</p>';
      },
    },
  ],
}).mount();
```

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `id` | `string` | ✓ | Unique panel identifier |
| `icon` | `string` | ✓ | Icon name from the SVG sprite |
| `title` | `string` | ✓ | Tab label |
| `render` | `function` | ✓ | Called with `(container, { map, instanceId })` to populate content |

> **Buttons vs Panels:** Use `buttons` when you want a toolbar button (optionally with a panel). Use `panels` when you only need a panel tab. Both can be combined in the same config.

See [Extending — Custom Panels](./8.extending.md#custom-panels) for more details.

---

## `plugins`

**Type:** `array` **Default:** `[]`

Installs per-instance plugins. Each plugin is an object with an `install` method:

```js
Navigator.create({
  id: "my-map",
  plugins: [MyPlugin],
}).mount();
```

For global plugins that apply to every instance, use `Navigator.use()` before creating instances. See [Extending — Plugin System](./8.extending.md#plugin-system) for full details.

## Lifecycle Callbacks

Convenience callbacks that map directly to events on the per-instance event emitter.

```js
Navigator.create({
  id: "my-map",
  onMapReady: ({ map, instanceId }) => {
    /* map loaded */
  },
  onViewChange: ({ center, zoom }) => {
    /* map moved */
  },
  onThemeChange: (theme) => {
    /* 'light' or 'dark' */
  },
  onPanelChange: (panelId) => {
    /* panel tab switched */
  },
}).mount();
```

See [Extending — Lifecycle Callbacks](./8.extending.md#lifecycle-callbacks) for payload details and the equivalent event emitter API.
