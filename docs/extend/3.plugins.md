# Plugins

Plugins are reusable extensions that receive the Navigator context and can register event listeners, provide data to the Vue tree, or add map layers. They are the recommended way to build self-contained features.

---

## Writing a Plugin

A plugin is any object with an `install` method. The `install` method may optionally return a **cleanup function** that will be called when the Navigator instance is unmounted:

```js
const MarkerPlugin = {
  install({ instanceId, on, once, emit, addButton }, options) {
    // Listen for map ready (fires once, so use once())
    once('map:ready', ({ map }) => {
      map.addSource('markers', {
        type: 'geojson',
        data: options.data,
      });
      map.addLayer({
        id: 'markers-layer',
        type: 'circle',
        source: 'markers',
        paint: { 'circle-radius': 6, 'circle-color': '#e74c3c' },
      });
      emit('markers:loaded', { count: options.data.features.length });
    });

    // Register a toolbar button from within the plugin
    addButton({
      id: 'markers',
      icon: 'position',
      label: 'Markers',
    });

    // Return a cleanup function (optional)
    return () => {
      // Called when the Navigator instance is unmounted
    };
  },
};
```

## Plugin Context

The `install` function receives a context object and optional options:

| Property | Description | Since |
|----------|-------------|-------|
| `app` | The Vue 3 `App` instance | 1.0.0 |
| `instanceId` | The Navigator instance ID | 1.0.0 |
| `map` | `shallowRef` — `null` until MapLibre loads, then the `Map` instance. Use with `watch()` for reactive map access | 1.0.23 |
| `useStorage(ns, defaults)` | Pre-scoped reactive localStorage — no `instanceId` needed | 1.0.23 |
| `useSettings()` | Pre-scoped user preferences — returns `{ resolvedTheme, isDark, isMetric, resolvedUnits, toggleTheme, setUnits, language, setLanguage }` | 1.0.24 |
| `useLocale()` | Pre-scoped locale/translations — returns `{ locale, locales, localeNames, mapLanguageTag, t, setLocale }` | 1.0.24 |
| `getMap()` | Returns the current MapLibre `Map` instance or `null` — no `instanceId` needed | 1.0.23 |
| `onMapReady(callback)` | Called with `{ map, addSource, addLayer }` when MapLibre loads. Sources/layers added via `addSource`/`addLayer` are **auto-removed** on unmount | 1.0.23 |
| `on(event, fn)` | Register an event listener | 1.0.0 |
| `once(event, fn)` | Register a one-time event listener | 1.0.21 |
| `off(event, fn)` | Remove an event listener | 1.0.0 |
| `emit(event, ...args)` | Emit an event | 1.0.0 |
| `provide(key, value)` | Shorthand for `app.provide(key, value)` — share data with Vue components via `inject()` | 1.0.22 |
| `addButton(config)` | Register a custom toolbar button (same shape as `buttons` config entries) | 1.0.22 |
| `addPanel(config)` | Register a custom panel tab (same shape as `panels` config entries) | 1.0.22 |

If `install` returns a function, it will be called when the Navigator instance is unmounted (see [Plugin Cleanup](#plugin-cleanup) below).

## Global Plugins

Register a plugin globally so it is installed on every instance:

```js
Navigator.use(MarkerPlugin, { data: myGeoJSON });

// Both instances receive the plugin
Navigator.create({ id: 'map-a' }).mount();
Navigator.create({ id: 'map-b' }).mount();
```

`Navigator.use()` returns `Navigator` for chaining:

```js
Navigator
  .use(MarkerPlugin, { data: pointsGeoJSON })
  .use(AnalyticsPlugin, { trackingId: 'UA-XXX' });
```

## Per-Instance Plugins

Pass plugins in the config to install only on a specific instance:

```js
Navigator.create({
  id: 'map-a',
  plugins: [
    { install: ({ on }) => on('map:ready', () => console.log('map-a ready')) },
  ],
}).mount();
```

### Per-instance plugin options

Global plugins receive options via `Navigator.use(plugin, options)`. Per-instance plugins support two equivalent syntaxes:

**Tuple syntax** (array of `[plugin, options]`):

```js
Navigator.create({
  id: 'map-a',
  plugins: [
    [RecordingsPlugin, { maxDuration: 3600 }],
  ],
}).mount();
```

**Object syntax** (`{ plugin, options }`):

```js
Navigator.create({
  id: 'map-a',
  plugins: [
    { plugin: RecordingsPlugin, options: { maxDuration: 3600 } },
  ],
}).mount();
```

Both forms pass `options` as the second argument to `install()`, just like `Navigator.use()` does for global plugins.

## Plugin Cleanup

Plugins can clean up resources (timers, geolocation watches, map layers) in several ways:

**1. Auto-cleanup via `onMapReady()` (recommended for map layers)**

Sources and layers registered via `onMapReady`'s `addSource`/`addLayer` helpers are automatically removed when the instance unmounts — no manual teardown needed:

```js
const LayerPlugin = {
  install({ onMapReady }) {
    onMapReady(({ addSource, addLayer }) => {
      addSource('my-source', { type: 'geojson', data: emptyGeoJSON });
      addLayer({ id: 'my-layer', type: 'circle', source: 'my-source' });
      // Automatically removed on unmount
    });
  },
};
```

**2. Return a cleanup function from `install()`**

The returned function is called automatically when the Navigator instance is unmounted:

```js
const PollingPlugin = {
  install() {
    const timerId = setInterval(() => { /* poll */ }, 5000);

    // Cleanup: called on unmount
    return () => clearInterval(timerId);
  },
};
```

**3. Listen for the `destroy` event**

The `destroy` event fires immediately before the instance unmounts. This is useful when the cleanup logic is set up after install (e.g. inside a `map:ready` handler):

```js
const LayerPlugin = {
  install({ on, once }) {
    let map = null;

    once('map:ready', ({ map: m }) => {
      map = m;
      map.addSource('my-source', { type: 'geojson', data: emptyGeoJSON });
      map.addLayer({ id: 'my-layer', type: 'circle', source: 'my-source' });
    });

    on('destroy', () => {
      if (map?.getLayer('my-layer')) map.removeLayer('my-layer');
      if (map?.getSource('my-source')) map.removeSource('my-source');
    });
  },
};
```

All mechanisms can be combined. The `destroy` event fires first, then auto-cleanup removes tracked sources/layers, then return-value cleanup functions run, then the Vue app unmounts.

## Build Prerequisites

When using Vue Single File Components (`.vue` files) in your plugins, buttons, or panels, your Vite config must include `@vitejs/plugin-vue`:

```bash
npm install --save-dev @vitejs/plugin-vue
```

```js
// vite.config.js
import vue from '@vitejs/plugin-vue';

export default {
  plugins: [vue()],
};
```

Without this plugin, Vite cannot compile `.vue` files and the build will fail with an opaque transform error. If you only use plain `.js` plugins (no SFCs), this is not needed.
